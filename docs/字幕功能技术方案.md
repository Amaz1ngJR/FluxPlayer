# FluxPlayer 字幕功能技术方案

## 一、需求概述

- 通过配置项控制字幕开关
- 媒体文件/流中有字幕流时，直接解码并渲染
- 无字幕流时，预留 ASR 接口（whisper.cpp，后续实现）

---

## 二、难度评估

| 功能 | 难度 | 说明 |
|------|------|------|
| 内嵌字幕流解码渲染 | ★★★☆☆ | FFmpeg 已有完整 subtitle API，主要工作在渲染和时间同步 |
| 实时 ASR 字幕（whisper.cpp） | ★★★★☆ | 需要集成第三方库、管理模型文件、处理延迟和线程同步 |

---

## 三、整体架构

### 新增模块

```
src/subtitle/
├── SubtitleDecoder.h/.cpp    # FFmpeg 字幕解码（需 #include "FluxPlayer/pch.h"）
├── SubtitleManager.h/.cpp    # 字幕时间管理，查询当前应显示的字幕（需 #include "FluxPlayer/pch.h"）
└── (ASRSubtitleProvider.h/.cpp)  # 预留 ASR 接口，后续实现
```

### 数据流

```
Demuxer（字幕包）
    ↓
SubtitleDecoder::decode()
    ↓
SubtitleManager（带时间戳的字幕条目队列）
    ↓
Controller::renderSubtitles()（ImGui 渲染，每帧查询当前 PTS）
```

### 与现有架构的集成点

| 位置 | 改动 |
|------|------|
| `Demuxer` | 新增 `m_subtitleStreamIndex`，`open()` 中检测字幕流，`readPacket()` 路由字幕包 |
| `Player` | 新增 `subtitleDecoder_`、`subtitleManager_` 成员；解码线程处理字幕包 |
| `Controller` | 新增 `renderSubtitles()` 方法，每帧从 SubtitleManager 查询并渲染 |
| `Config::Settings` | 新增字幕相关配置项 |

## 四、内嵌字幕流方案

### 4.1 Demuxer 扩展

```cpp
// Demuxer.h 新增
int getSubtitleStreamIndex() const { return m_subtitleStreamIndex; }
AVStream* getSubtitleStream() const;
AVCodecParameters* getSubtitleCodecParams() const;

// 私有成员
int m_subtitleStreamIndex = -1;
```

`open()` 中在流遍历循环里增加：
```cpp
} else if (codecType == AVMEDIA_TYPE_SUBTITLE && m_subtitleStreamIndex == -1) {
    m_subtitleStreamIndex = i;
}
```

`readPacket()` 返回的 packet 已携带 `stream_index`，Player 侧按 index 路由即可，Demuxer 无需改动读取逻辑。

### 4.2 SubtitleDecoder

```cpp
class SubtitleDecoder {
public:
    bool init(AVCodecParameters* codecParams, AVRational timeBase);
    // 返回 true 表示解码出字幕，subtitle 需调用 avsubtitle_free 释放
    bool decode(AVPacket* packet, AVSubtitle& subtitle, double& startPTS, double& endPTS);
    void flush();
    void close();
private:
    AVCodecContext* m_codecCtx = nullptr;
    AVRational m_timeBase;
};
```

FFmpeg 字幕解码 API：
```cpp
int gotSub = 0;
avcodec_decode_subtitle2(m_codecCtx, &subtitle, &gotSub, packet);
// startPTS = packet->pts * av_q2d(m_timeBase)
// endPTS   = startPTS + subtitle.end_display_time / 1000.0
```

### 4.3 支持的字幕格式

| 格式 | 类型 | FFmpeg 解码器 | 说明 |
|------|------|--------------|------|
| SRT / SubRip | 文本 | `subrip` | 最常见，纯文本 |
| ASS / SSA | 文本 | `ass` | 支持样式，MKV 常见 |
| WebVTT | 文本 | `webvtt` | HTML5 标准 |
| MOV_TEXT | 文本 | `mov_text` | MP4 内嵌字幕 |
| PGS / HDMV | 图形位图 | `pgssub` | 蓝光字幕，阶段二支持 |
| DVB-SUB | 图形位图 | `dvbsub` | 广播字幕，阶段二支持 |

**阶段一只处理文本类字幕**（`subtitle.rects[i]->type == SUBTITLE_ASS` 或 `SUBTITLE_TEXT`），图形字幕跳过。

### 4.4 SubtitleManager

```cpp
struct SubtitleEntry {
    std::string text;   // 纯文本（去除 ASS 标签后）
    double startPTS;
    double endPTS;
};

class SubtitleManager {
public:
    void addEntry(const SubtitleEntry& entry);
    // 根据当前播放 PTS 返回应显示的字幕，无则返回空字符串
    std::string getCurrentText(double currentPTS) const;
    void clear();
private:
    mutable std::mutex m_mutex;
    std::deque<SubtitleEntry> m_entries;  // 按 startPTS 有序
};
```

`getCurrentText` 逻辑：遍历 `m_entries`，找到 `startPTS <= currentPTS <= endPTS` 的条目返回其文本；同时清理已过期（`endPTS < currentPTS - 5.0`）的旧条目。

### 4.5 时间同步策略

- 字幕 PTS 来自 `AVPacket::pts * av_q2d(subtitleStream->time_base)`
- 渲染时从 `avSync_->getAudioClock()`（或视频时钟）获取当前 PTS
- 直接比较，无需额外补偿
- Seek 后调用 `SubtitleManager::clear()` 清空缓存，`SubtitleDecoder::flush()` 刷新解码器

### 4.6 ASS 标签清理

ASS 格式的文本包含 `{\an8}{\pos(320,50)}` 等控制标签，需简单清理：
```cpp
// 去除 {…} 包裹的 ASS 控制标签
std::string stripASSOverrides(const std::string& ass) {
    std::string result;
    bool inTag = false;
    for (char c : ass) {
        if (c == '{') inTag = true;
        else if (c == '}') inTag = false;
        else if (!inTag) result += c;
    }
    return result;
}
```

## 五、字幕渲染方案

### 5.1 ImGui 文本渲染

在 `Controller::render()` 末尾调用 `renderSubtitles()`：

```cpp
void Controller::renderSubtitles() {
    if (!subtitleEnabled_ || !subtitleManager_) return;
    double pts = player_.getCurrentTime();
    std::string text = subtitleManager_->getCurrentText(pts);
    if (text.empty()) return;

    ImGuiIO& io = ImGui::GetIO();
    float winW = io.DisplaySize.x;
    float winH = io.DisplaySize.y;

    ImGui::SetNextWindowPos(ImVec2(winW * 0.5f, winH - 80.0f), ImGuiCond_Always, ImVec2(0.5f, 1.0f));
    ImGui::SetNextWindowBgAlpha(0.55f);
    ImGui::SetNextWindowSize(ImVec2(winW * 0.85f, 0), ImGuiCond_Always);
    ImGuiWindowFlags flags = ImGuiWindowFlags_NoDecoration | ImGuiWindowFlags_NoInputs |
                             ImGuiWindowFlags_NoNav | ImGuiWindowFlags_NoMove |
                             ImGuiWindowFlags_NoSavedSettings | ImGuiWindowFlags_NoBringToFrontOnFocus;
    ImGui::Begin("##subtitle", nullptr, flags);
    ImGui::SetWindowFontScale(subtitleFontScale_);
    ImGui::TextWrapped("%s", text.c_str());
    ImGui::End();
}
```

### 5.2 多行字幕

`TextWrapped` 自动换行，窗口宽度限制为屏幕 85%，高度自适应内容。

### 5.3 图形字幕（PGS/DVB，阶段二）

- 将 `AVSubtitleRect` 的位图数据上传为 OpenGL RGBA 纹理（使用 `unsigned int` 纹理句柄，与 GLRenderer 保持一致）
- 在 GLRenderer 中叠加渲染（类似 OSD），注意 NV12 渲染现有 `m_nv12Deinterleave` 两种模式（GL_RG8 零拷贝 vs UV ��交错兼容模式），图形字幕纹理独立于此
- 阶段一跳过，仅记录 TODO

## 七、Player 集成

### 7.1 新增成员

```cpp
// Player.h 私有成员
std::unique_ptr<SubtitleDecoder> subtitleDecoder_;
std::unique_ptr<SubtitleManager> subtitleManager_;
```

### 7.2 initDecoders() 中初始化

```cpp
if (demuxer_->getSubtitleStreamIndex() >= 0 && config.subtitleEnabled) {
    subtitleDecoder_ = std::make_unique<SubtitleDecoder>();
    subtitleDecoder_->init(demuxer_->getSubtitleCodecParams(),
                           demuxer_->getSubtitleStream()->time_base);
    subtitleManager_ = std::make_unique<SubtitleManager>();
}
```

### 7.3 解码线程路由字幕包

```cpp
// decodingThread() 中 readPacket 后
if (packet->stream_index == demuxer_->getSubtitleStreamIndex() && subtitleDecoder_) {
    AVSubtitle sub;
    double startPTS, endPTS;
    if (subtitleDecoder_->decode(packet, sub, startPTS, endPTS)) {
        for (unsigned i = 0; i < sub.num_rects; i++) {
            if (sub.rects[i]->type == SUBTITLE_ASS || sub.rects[i]->type == SUBTITLE_TEXT) {
                std::string raw = sub.rects[i]->type == SUBTITLE_ASS
                    ? sub.rects[i]->ass : sub.rects[i]->text;
                subtitleManager_->addEntry({stripASSOverrides(raw), startPTS, endPTS});
            }
        }
        avsubtitle_free(&sub);
    }
}
```

### 7.4 Seek 时清理

```cpp
// seek() 中，刷新解码器后
if (subtitleDecoder_) subtitleDecoder_->flush();
if (subtitleManager_) subtitleManager_->clear();
```

### 7.5 暴露 SubtitleManager 给 Controller

```cpp
// Player.h 公开接口
SubtitleManager* getSubtitleManager() const { return subtitleManager_.get(); }
bool hasSubtitleStream() const { return subtitleDecoder_ != nullptr; }
```

## 十、风险与应对

| 风险 | 说明 | 应对 |
|------|------|------|
| ASS 标签解析不完整 | 复杂 ASS 样式标签清理不干净 | 阶段一只做基础 `{…}` 去除，复杂样式忽略 |
| 字幕时间偏移 | 部分文件字幕 PTS 与视频不对齐 | 预留 `subtitleOffset` 配置项（秒），默认 0 |
| 图形字幕（PGS）跳过 | 阶段一不支持 | 检测到图形字幕时在 UI 提示"不支持的字幕格式" |
| 多字幕流选择 | 文件含多语言字幕流 | 阶段一默认选第一条，后续在设置菜单增加轨道选择 |
| ImGui 中文字体 | 默认字体不含中文字符 | 需加载支持 CJK 的字体（如 NotoSansCJK），或提示用户配置字体路径 |

## 十一、验证方式

1. 用含 SRT/ASS 字幕的 MKV 文件测试：字幕正确显示、时间同步
2. 用无字幕的 MP4 测试：不崩溃，字幕区域为空
3. Seek 后字幕正确更新，无残留旧字幕
4. 配置 `subtitleEnabled=false` 后字幕不显示
5. 用 RTMP 实时流测试：无字幕流时正常播放
