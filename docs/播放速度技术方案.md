# FluxPlayer 播放速度控制技术方案

## 一、需求概述

- 支持 0.5x、0.75x、1.0x、1.25x、1.5x、2.0x 六档播放速度
- 快放：外部时钟加速驱动视频帧显示，智能丢帧（保留 I 帧，优先丢 B 帧）
- 慢放：外部时钟减速驱动视频帧显示
- 音频：最近邻重采样变速（快放音调升高，慢放音调降低）
- UI：控制栏速度选择弹出菜单，实时切换
- 无需引入外部依赖，纯 C++ 实现

## 三、时钟驱动机制

FluxPlayer 在有音频输出时使用 `EXTERNAL_CLOCK`（系统时钟）驱动视频同步。变速的核心是让外部时钟以 `rate` 倍实时速率推进：

```cpp
// AVSync::getExternalClock()
double elapsed = duration_cast<double>(now - externalClockBase_).count();
return externalClockOffset_ + elapsed * playbackRate_;
```

**速率切换时防时钟跳变**：切换速率前先锁定当前时钟值，重置基准：

```cpp
// AVSync::setPlaybackRate(rate)
double current = getExternalClock();
externalClockBase_ = steady_clock::now();
externalClockOffset_.store(current);
playbackRate_.store(rate);
```

若不重置基准，切换后 `elapsed × newRate` 会导致时钟大幅跳变（如从 2x 切到 0.5x 时时钟跳退，视频冻结）。

## 四、快放丢帧策略（1.25x ~ 2.0x）

外部时钟加速后，视频帧会被快速消耗。为减少解码压力，对已取出的帧做智能丢帧：

| 帧类型 | 策略 |
|--------|------|
| I 帧（关键帧） | 永不丢弃（解码基准） |
| B 帧 | 按概率丢弃：`(rate-1)/rate`（1.25x→20%，2.0x→50%） |
| P 帧 | 仅 2.0x 时以 25% 概率丢弃 |

```cpp
// Player::shouldDropFrameForSpeed()
if (avFrame->key_frame) return false;
if (avFrame->pict_type == AV_PICTURE_TYPE_B) {
    uint64_t threshold = (rate - 1.0) / rate * 100;
    return (frameDropCounter_++ % 100) < threshold;
}
if (avFrame->pict_type == AV_PICTURE_TYPE_P && rate >= 2.0)
    return (frameDropCounter_++ % 4) == 0;
return false;
```

## 五、音频变速重采样

### 原理

通过改变每次回调消耗的音频内容量实现变速：

- **2.0x**：消耗 2× 内容，重采样压缩到 1× 输出 → 音调升高一倍
- **0.5x**：消耗 0.5× 内容，重采样拉伸到 1× 输出 → 音调降低一半
- **1.0x**：直接输出，无重采样

### 实现

```cpp
// audioOutputCallback 核心逻辑
size_t inputNeeded = bufferSize * rate;  // 需消耗的内容字节数

// 1. 从队列读取 inputNeeded 字节到临时缓冲区
std::vector<uint8_t> inputBuf(inputNeeded, 0);
// ... 从 audioQueue_ 读取 ...

// 2. 最近邻重采样：inputFilled 字节 → bufferSize 字节
for (int i = 0; i < outSamples; i++) {
    int srcIdx = i * inSamples / outSamples;
    memcpy(buffer + i * frameBytes, inputBuf + srcIdx * frameBytes, frameBytes);
}

// 3. 音频时钟按速率推进
currentAudioPTS = prevClock + deviceDuration * rate;
```

### 音频队列管理

快放时消耗更多音频内容，音频队列自然加速消耗，解码线程不会因队列满而阻塞视频帧产出。慢放时消耗减少，队列自然积累，无需额外处理。

## 六、UI 速度选择器

在底部控制栏设置图标左侧添加速度��钮（"Spd"），使用 ImGui Popup 实现弹出菜单：

```
底部控制栏：
[Play] [Stop] [Rec]  ...  [Spd] [⚙] [🔊][===]
                            ↓ 点击弹出
                     ┌─────────────┐
                     │  0.5x       │
                     │  0.75x      │
                     │ ● 1.0x      │  ← 当前选中（蓝色）
                     │  1.25x      │
                     │  1.5x       │
                     │  2.0x       │
                     └─────────────┘
```

- 非 1.0x 时按钮高亮（蓝色），并在按钮下方显示当前速率（如 "2x"）
- 使用 `ImGui::OpenPopup` / `BeginPopup`，点击外部自动关闭

## 七、配置项

### fluxplayer.ini

```ini
[Speed]
# 说明：默认播放速度倍率
# 取值：0.5 / 0.75 / 1.0 / 1.25 / 1.5 / 2.0
# 默认：1.0
playbackSpeed=1.0

# 说明：慢放时是否启用帧插值（当前版本暂未集成到渲染路径）
# 取值：true / false
# 默认：true
frameInterpolation=true
```

### Config::Settings 新增字段

```cpp
double playbackSpeed = 1.0;      // 默认播放速度
bool frameInterpolation = true;  // 慢放帧插值开关（预留）
```

## 八、性能分析

| 操作 | 耗时 | 说明 |
|------|------|------|
| 外部时钟速率调整 | ~0ns | 仅乘法运算 |
| 丢帧判断 | ~0ns | 仅读取帧类型标志 |
| 最近邻重采样（4096 样本） | <0.1ms | 简单索引映射 |
| 时钟基准重置（速率切换） | ~0ns | 原子操作 |

## 九、已知限制

| 场景 | 当前行为 |
|------|---------|
| 音调保持 | 不支持（快放音调升高，慢放音调降低） |
| 慢放帧插值 | FrameInterpolator 已实现但未集成到渲染路径 |
| 实时流 | 未特殊处理，慢放可能导致队列积压 |
| 纯音频文件 | 仅音频变速，无视频逻辑 |
